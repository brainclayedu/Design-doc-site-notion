Attack Types

Goals

The attack types system aims to:

  • Define Combat Roles: Establish five distinct categories to shape attack interactions, enhancing tactical variety.
  • Support Core Loop: Reinforce rock-paper-scissors dynamics with clear counters and synergies, fluid in 1v1 and group play.
  • Enable Scalability: Ensure categories adapt to multi-target scenarios, supporting summons and future MMORPG modes.
  • Blend Genres: Merge fighting game precision with RPG depth, integrating with states, weapons, and gauges.

Overview

Attack Types in The Fighting Game categorize all combat actions into five core types—Strike, Projectile, Beam, Grab, control skill—each governing how attacks interact in a 3rd-person over-the-shoulder perspective. Strikes deliver standard blows, Projectiles fire independent entities, Beams channel continuous effects, Grabs seize foes up close, and control skills disrupt at range. Built for basic weapons (e.g., longsword Strikes, spear Throws), these types support skill-based hybrids (e.g., Beam as a physical stream or magical blast), keeping gear simple for prototyping. Inherent AoE with friendly fire drives group tactics, while secondary traits (DoTs, Channeled) deepen strategy. Detailed clash mechanics—Strikes nullifying Strikes, Projectiles clashing by properties, Beams wagering Poise—add tactical layers, tied to states and gauges to power the core loop’s Prediction and Advantage, scaling from duels to chaotic arenas.

Core Attack Types System

Five categories define attack interactions, with detailed clash mechanics where applicable:

  • Strike: Standard attacks (e.g., punches, sword slashes), mid-to-long-range (non-projectile). Clash with Strikes, lose to Grabs, interruptible by control skills. Aim locked post-startup, turn-restricted.
    • Strike Clash: Occurs when disjointed hitboxes overlap (only clashing players affected, others hit normally). Nullifies the hit, 17-frame hitstop, 10% Poise gain, cancels into: any attack; jump (if jump-cancelable); forward run (ground) or forward air dash (air, if air option available). No air jump/dash cancel if prior air option used. Enables Faultless Defense baiting or retaliation.
  • Projectile: Independent entities (e.g., arrows, magical orb, magical fireball ), clash by level/property (solid/energy), shorter hitstun, aim tunable in startup, rare redirection mid-flight. Beat blocks, lose to control skills or deflecting Strikes; could lose to Beams (pending testing).
    • Projectile Clash: Determined by attack level (e.g., Lv.1 vs. Lv.3) and property (solid vs. energy)—higher level or specific counters (e.g., solid beats energy) win, nullifying weaker Projectile. Some Strikes nullify or deflect Projectiles instead.
  • Beam: Channeled attacks (e.g., sustained streams, physical or magical), continuous damage with hitstun buildup (triggers after delay, decays off-beam), blockable (deflects nearby, Faultless Defense extends range), no initial hitstun, blockstun builds slower. Focus/Impact-only, redirectable with character orientation turn limits; could overpower Projectiles (pending testing). Destroys some Projectiles.
    • Beam Clash: Occurs when beams intersect. Full clash if hurtbox/hitbox overlap both players; deflection if not aimed at opponent’s hitbox. Light vs. Heavy: Heavy pushes Light back (travel time allows reaction). Equal clash: Tap combo key to bid 1 Poise block per press (max 5 blocks), highest bid wins, loser takes stored damage from clash phase, both lose bid Poise. Opt-out by releasing beam enters recovery, risking opponent’s stored damage hit.
  • Grab: Proximity-based, unblockable if no invuln and in range, shifts foes to a controlled state. Beats guards, loses to Strikes/control skills, directional (Base) or aimed (Impact). No clash mechanic defined.
  • control skill: Long-range crowd control skill (e.g., disarm knocks weapon 5-10m), high Energy/Poise cost (varies—quick cast blockable, long startup unblockable, AoE dodgeable), interruptible by Strikes/Grabs. Beats Projectiles, counters blocks with disabling effects (e.g., 1-3s stun). No clash mechanic defined.
  • Secondary Traits: Optional across types—AoE (all attacks, explicit like conical spikes), DoTs (stacking, boosts hitstun, e.g., bleeding Slash), Channeled (redirectable, charged, e.g., Beam or Projectile streams).

Integration with Combat

Attack types weave into the combat ecosystem:

  • States: Strikes favor Base (directional), Grabs/control skills use Impact (disruption), Projectiles/Beams suit Focus (precision), Guard counters via blocks. Disarmed defaults to weak Strikes unless custom fists slotted.
  • Gauges: Energy scales by type (e.g., control skills 20%, Beams 15%), Poise fuels Beam clashes (1 block/tap, max 5) and controls effects (e.g., 1-2 blocks). Higher costs balance power.
  • Core Loop: Strikes test Neutral (clash baiting), Grabs exploit Prediction, control skills shift Advantage (CC), Beams pressure Disadvantage (buildup), Projectiles probe spacing. AoE/friendly fire heightens group play.
  • Weapons: Longswords boost Strikes, shortbows fire Projectiles—fists default to Strikes if disarmed. Skills allow hybrids (e.g., Strike + Projectile dagger toss).
  • Scalability: control skills/Beams hit multiple foes, Projectiles pierce lines, Strikes sweep arcs—clashes scale to summons or 2v2 (e.g., Strike clash nullifies one foe’s hit).

Challenges and Solutions

  • Clash Complexity: Beam’s Poise bidding risks overcomplication.

    Solution: 1 block per tap (max 5, instant resolve) simplifies execution, retaining wagering; deeper mechanics (e.g., timing) testable later.

Conclusions

The attack types system delivers a robust framework—Strike, Projectile, Beam, Grab, control skill—blending fighting game precision with RPG depth. Detailed clashes (e.g., Strike’s 17-frame nullification, Beam’s Poise bid) and secondary traits (e.g., DoTs, AoE) enhance strategy, integrating with states, weapons, and gauges. Built for prototyping with basic weapons (e.g., spear Grabs) and skill-based hybrids (e.g., Beam as physical or magical), it scales to group combat with short, fun CC (1-3s), preserving original mechanics while supporting the core loop’s dynamic flow and future complexity.
