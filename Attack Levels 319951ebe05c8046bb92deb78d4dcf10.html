Attack Levels

Goals

The attack levels system aims to:

  • Standardize Hit Effects: Define tiers for hitstun, blockstun, and hitstop, ensuring consistent combat pacing.
  • Support Customization: Allow build overrides for tactical variety.
  • Scale with Combat: Enable flexibility from 1v1 to group scenarios with unbounded hits.
  • Integrate Fluidly: Tie to attack types and defensive mechanics.

Overview

Attack Levels in The Fighting Game assign each attack a tier (Lv.0-Lv.4) to standardize hit effects—such as hitstun, blockstun, hitstop, or control duration—in a 3rd-person over-the-shoulder perspective. These levels provide a baseline for combat pacing across all attack types, with values customizable via stats, gear, skills, and trees, ensuring predictable yet adaptable interactions within the core loop.

Integration with Combat

Levels tie into the ecosystem:

  • Attack Types: Strikes use full table, Projectiles lower stun, Beams buildup-based, Grabs/Controls apply CC (e.g., Lv.3 Control = 2s Fear). Priority via levels (e.g., Projectile Clash).
  • Weapons: Longswords (Slash) at Lv.2 = 16f hitstun—skills tweak via "Attack Properties" (e.g., Fire Lv.3).
  • Core Loop: Lv.0 tests Neutral (quick recovery), Lv.4 locks Advantage (long control)—air action restore aids Disadvantage escape. Hits unbounded for group play.
  • Guard: FD/IFD values from Guilty Gear Strive—prototyping to refine air penalties.

Challenges and Solutions

  • Customization Balance: Overrides risk imbalance (e.g., excessive stun).

    Solution: Moderate range via skill tree, test in prototyping.

  • Air Block Penalty: "Until Landing+19" harsh vs. ground 18f.

    Solution: Retain as Strive-inspired trade-off, adjustable post-testing.

  • Type Variation: Diverse tables need balance.

    Solution: Prototype Shoto moveset (e.g., Lv.2 fireball) to tune.

Conclusions

Attack Levels standardize combat pacing with Lv.0-Lv.4 tiers across all attack types, balancing consistency with build flexibility for a Shoto-style prototype (e.g., balanced archetype). Full tables for Strikes, Projectiles, Beams, Grabs, and Controls—unbounded for group hits—integrate with "Attack Types" and "Guard," scaling via friendly fire deterrence while supporting the core loop’s dynamic flow.
