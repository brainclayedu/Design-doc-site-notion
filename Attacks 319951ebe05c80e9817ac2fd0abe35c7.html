Attacks

Attack Types

Goals

The attack types system aims to:

  • Define Combat Roles: Establish five distinct categories to shape attack interactions, enhancing tactical variety.
  • Support Core Loop: Reinforce rock-paper-scissors dynamics with clear counters and synergies, fluid in 1v1 and group play.
  • Enable Scalability: Ensure categories adapt to multi-target scenarios, supporting summons and future MMORPG modes.
  • Blend Genres: Merge fighting game precision with RPG depth, integrating with states, weapons, and gauges.

Overview

Attack Types in The Fighting Game categorize all combat actions into five core types—Strike, Projectile, Beam, Grab, control skill—each governing how attacks interact in a 3rd-person over-the-shoulder perspective. Strikes deliver standard blows, Projectiles fire independent entities, Beams channel continuous effects, Grabs seize foes up close, and control skills disrupt at range. Built for basic weapons (e.g., longsword Strikes, spear Throws), these types support skill-based hybrids (e.g., Beam as a physical stream or magical blast), keeping gear simple for prototyping. Inherent AoE with friendly fire drives group tactics, while secondary traits (DoTs, Channeled) deepen strategy. Detailed clash mechanics—Strikes nullifying Strikes, Projectiles clashing by properties, Beams wagering Poise—add tactical layers, tied to states and gauges to power the core loop’s Prediction and Advantage, scaling from duels to chaotic arenas.

Core Attack Types System

Five categories define attack interactions, with detailed clash mechanics where applicable:

  • Strike: Standard attacks (e.g., punches, sword slashes), mid-to-long-range (non-projectile). Clash with Strikes, lose to Grabs, interruptible by control skills. Aim locked post-startup, turn-restricted.
    • Strike Clash: Occurs when disjointed hitboxes overlap (only clashing players affected, others hit normally). Nullifies the hit, 17-frame hitstop, 10% Poise gain, cancels into: any attack; jump (if jump-cancelable); forward run (ground) or forward air dash (air, if air option available). No air jump/dash cancel if prior air option used. Enables Faultless Defense baiting or retaliation.
  • Projectile: Independent entities (e.g., arrows, magical orb, magical fireball ), clash by level/property (solid/energy), shorter hitstun, aim tunable in startup, rare redirection mid-flight. Beat blocks, lose to control skills or deflecting Strikes; could lose to Beams (pending testing).
    • Projectile Clash: Determined by attack level (e.g., Lv.1 vs. Lv.3) and property (solid vs. energy)—higher level or specific counters (e.g., solid beats energy) win, nullifying weaker Projectile. Some Strikes nullify or deflect Projectiles instead.
  • Beam: Channeled attacks (e.g., sustained streams, physical or magical), continuous damage with hitstun buildup (triggers after delay, decays off-beam), blockable (deflects nearby, Faultless Defense extends range), no initial hitstun, blockstun builds slower. Focus/Impact-only, redirectable with character orientation turn limits; could overpower Projectiles (pending testing). Destroys some Projectiles.
    • Beam Clash: Occurs when beams intersect. Full clash if hurtbox/hitbox overlap both players; deflection if not aimed at opponent’s hitbox. Light vs. Heavy: Heavy pushes Light back (travel time allows reaction). Equal clash: Tap combo key to bid 1 Poise block per press (max 5 blocks), highest bid wins, loser takes stored damage from clash phase, both lose bid Poise. Opt-out by releasing beam enters recovery, risking opponent’s stored damage hit.
  • Grab: Proximity-based, unblockable if no invuln and in range, shifts foes to a controlled state. Beats guards, loses to Strikes/control skills, directional (Base) or aimed (Impact). No clash mechanic defined.
  • control skill: Long-range crowd control skill (e.g., disarm knocks weapon 5-10m), high Energy/Poise cost (varies—quick cast blockable, long startup unblockable, AoE dodgeable), interruptible by Strikes/Grabs. Beats Projectiles, counters blocks with disabling effects (e.g., 1-3s stun). No clash mechanic defined.
  • Secondary Traits: Optional across types—AoE (all attacks, explicit like conical spikes), DoTs (stacking, boosts hitstun, e.g., bleeding Slash), Channeled (redirectable, charged, e.g., Beam or Projectile streams).

Integration with Combat

Attack types weave into the combat ecosystem:

  • States: Strikes favor Base (directional), Grabs/control skills use Impact (disruption), Projectiles/Beams suit Focus (precision), Guard counters via blocks. Disarmed defaults to weak Strikes unless custom fists slotted.
  • Gauges: Energy scales by type (e.g., control skills 20%, Beams 15%), Poise fuels Beam clashes (1 block/tap, max 5) and controls effects (e.g., 1-2 blocks). Higher costs balance power.
  • Core Loop: Strikes test Neutral (clash baiting), Grabs exploit Prediction, control skills shift Advantage (CC), Beams pressure Disadvantage (buildup), Projectiles probe spacing. AoE/friendly fire heightens group play.
  • Weapons: Longswords boost Strikes, shortbows fire Projectiles—fists default to Strikes if disarmed. Skills allow hybrids (e.g., Strike + Projectile dagger toss).
  • Scalability: control skills/Beams hit multiple foes, Projectiles pierce lines, Strikes sweep arcs—clashes scale to summons or 2v2 (e.g., Strike clash nullifies one foe’s hit).

Challenges and Solutions

  • Clash Complexity: Beam’s Poise bidding risks overcomplication.

    Solution: 1 block per tap (max 5, instant resolve) simplifies execution, retaining wagering; deeper mechanics (e.g., timing) testable later.

Conclusions

The attack types system delivers a robust framework—Strike, Projectile, Beam, Grab, control skill—blending fighting game precision with RPG depth. Detailed clashes (e.g., Strike’s 17-frame nullification, Beam’s Poise bid) and secondary traits (e.g., DoTs, AoE) enhance strategy, integrating with states, weapons, and gauges. Built for prototyping with basic weapons (e.g., spear Grabs) and skill-based hybrids (e.g., Beam as physical or magical), it scales to group combat with short, fun CC (1-3s), preserving original mechanics while supporting the core loop’s dynamic flow and future complexity.

Clash Mechanics

Introduction

When two attacks collide on the same frame, the game enters a brief, high-tension moment called a Clash. This is one of the most satisfying Yomi moments in the combat system. You feel the impact, the screen flashes, both characters pause for a split second, and then the fight can swing wildly depending on who reacts faster. Clashes reward perfect timing and prediction, turning a simple trade into a mind-game opportunity.

Goals

  • Create exciting, high-stakes moments when attacks collide.
  • Reward precise timing and reads with Poise gain and cancel opportunities.
  • Keep the flow fast and fair — no one feels cheated by a clash.
  • Tie clashes into the broader resource and state systems.

Overview

Clash occurs when two attacks hit on the exact same frame. The game resolves the clash based on the type of attack:

  • Strike vs Strike = Strike Clash
  • Grab vs Grab = Grab Clash
  • Beam vs Beam = Beam Clash

All clashes share the same core effects:

  • Both attacks are nullified (no damage to either player).
  • Both characters enter hitstop for 17 frames.
  • Both gain 10% Poise.
  • Either player can cancel out of the clash into another move (Roman Cancel, Burst, etc.).

Core Mechanics

Toggle: Strike Clash
When two strikes (Light, Heavy, or Special) collide:

  • Both attacks are nullified.
  • 17-frame hitstop for both players.
  • 10% Poise gain for both.
  • Both players can immediately cancel into another move.

Toggle: Grab Clash
When two grabs (normal or special) collide:

  • Both grabs are nullified.
  • 17-frame hitstop.
  • 10% Poise gain.
  • The player who inputs the grab during the 10-frame window gains a slight advantage (priority in the next exchange).

Toggle: Beam Clash
When two beam-style supers collide:

  • Both players wager Poise (higher wager wins).
  • Winner’s beam continues; loser’s is nullified.
  • Winner gains extra Poise and Burst.

Integration with Other Systems

  • Poise: Clashes are one of the fastest ways to gain Poise, rewarding aggressive reads.
  • Roman Cancel / Burst: You can cancel out of a clash for big momentum swings.
  • Input Control Modes: Aggressive mode increases clash priority slightly.
  • States: Clashes can occur in any Combat State and often lead to Traversal State transitions (e.g., clash → dash).

Problems & Solutions

ProblemSolution
Clashes feel randomClear visual flash + hitstop + sound cue makes them obvious.
Too many clashes in group fightsDecay on repeated identical attacks reduces clash frequency.
New players miss the cancel windowTraining mode shows clash timing and cancel prompts.

Playtest & Iteration Notes

  • MVP Metrics: Clash frequency, Poise gain rate from clashes, cancel success rate.
  • Blueprint Notes: All clash rules (hitstop frames, Poise gain, priority) are exposed variables for easy tuning.

Scaling

Phase 0 (MVP): 1v1 strike/grab clashes.
MOBA/MMORPG: Beam clashes and team-clash priority for large-scale fights.

Conclusion

Clash mechanics turn potential trades into exciting mind-games. They reward precise timing, build Poise fast, and give both players a fair chance to turn the fight around — exactly what makes the combat system feel alive and fair.

Attack Properties

Goals

The attack properties system aims to:

  • Enhance Tactical Depth: Define damage types to enrich attack variety and suggest logical effects.
  • Support Combat Flow: Enable complementary and clashing interactions within the core loop.
  • Scale with Environment: Leverage player and arena effects for group combat and MMORPG potential.
  • Integrate Seamlessly: Blend with attack types and weapons, balancing complexity.

Overview

Attack Properties in The Fighting Game assign damage types to attacks, shaping their impact in a 3rd-person over-the-shoulder perspective. Split into Magic (e.g., Fire, Cold) and Physical (e.g., Pierce, Slash), properties suggest effects—Slash may bleed, Cold may slow—and enable interactions like Fire + Cold creating steam. Non-magical weapons (e.g., longsword) combine with skill-based Magic (e.g., Fire burst) for hybrid attacks, keeping gear simple for prototyping. Player effects (e.g., Bleeding) and arena hazards (e.g., steam clouds) emerge from attacks or interactions, tied to "Attack Types" (e.g., Control potency), scaling from 1v1 duels to multi-foe chaos with all 21 types listed, starting small (Fire, Cold, Pierce, Slash).

Core Attack Properties System

Properties define damage types and interactions:

  • Magic:
    • Elemental: Fire, Cold, Lightning.
    • Nature: Plant, Earth, Wind, Water, Thunder.
    • Divine: Divine, Radiant, Darkness, Life, Death.
    • Other: Force, Psychic, Sonic, Arcane, Time, Space.
    • Prototyping Core: Fire, Cold.
  • Physical: Pierce, Slash, Bludgeon, Bleeding, Poison, Acid.
    • Prototyping Core: Pierce, Slash.
  • Interactions: Logical type-based effects, refined via prototyping:
    • Fire + Cold: Steam cloud (arena hazard, blind chance on players).
    • Fire + Wind: Wider, moving AoE (fire spreads).
    • Fire vs. Cold: Nullify.
    • Cold + Pierce: builds up slow or freeze.
    • Cold + Slash: Slow energy regen.
    • Slash + poison: DoT Poison damage.
    • Complementary boosts (e.g., increased damage/AoE), clashing reduces (e.g., partial nullification)—exact values via skills, stats, gear.
  • Effects: Suggest logical outcomes—Slash/Pierce may bleed, Cold may slow, Fire may burn. Player effects (short duration) and arena effects (longer duration) applied via attacks or interactions, specifics (e.g., duration, stacking) determined by skill crafting, skill tree, stats, and gear.

Integration with Combat

Properties enhance the ecosystem via attacks:

  • Attack Types: Strikes carry Slash, Projectiles fire Cold, Control attacks apply Fear, Beams channel Fire—skills blend (e.g., Pierce + Cold builds up slow).
  • Weapons: Spears deal Pierce, longswords Slash—attack skills can be or can add Magic (e.g., Fire burst via spear). Disarmed reverts to Physical Strikes.
  • Core Loop: Fire tests Neutral (spacing), Slash exploits Prediction, Controls shift Advantage (CC), Cold pressures Disadvantage. Arena effects scale group play.
  • Scalability: Multi-foe hazards (e.g., Fire + Water = steam zone) suit summons/2v2.

Challenges and Solutions

  • Complexity: 21 types risk overwhelming prototyping.

    Solution: Start with Fire, Cold, Pierce, Slash; expand via skills, all listed for future.

  • Balance: Fire + Wind might overpower Cold.

    Solution: Cap boosts (e.g., moderate increase), tune reductions (e.g., balanced nullification), limit stacking via skill design.

  • Effect Clarity: Suggested vs. mandated effects could confuse.

    Solution: Skills define effects (e.g., Slash skill = Bleeding), properties suggest (e.g., Slash → Bleeding chance).

Conclusions

Attack Properties enrich combat with Magic (Elemental, Nature, Divine, etc.) and Physical types, suggesting effects (e.g., Slash → Bleeding) and blending logical interactions (e.g., Fire + Cold = steam) into the core loop. Prototyping-ready with non-magical weapons (e.g., spear Pierce) and skill-based hybrids (e.g., Cold slow), it scales via arena hazards, integrating with "Attack Types" while keeping gear simple and fun intact, with all 21 types poised for expansion.

Goals

  • Standardize Hit Effects: Define tiers for hitstun, blockstun, and hitstop, ensuring consistent combat pacing.

  • Integrate Fluidly: Tie to attack types and defensive mechanics.

Overview

Attack Levels in The Fighting Game assign each attack a tier (Lv.0-Lv.4) to standardize hit effects—such as hitstun, blockstun, hitstop, or control duration—in a 3rd-person over-the-shoulder perspective. These levels provide a baseline for combat pacing across all attack types, with values customizable via stats, gear, skills, and trees, ensuring predictable yet adaptable interactions within the core loop.

Attack and damage Priority

Goals

The attack and damage priority system aims to:

  • Encourage Strategic Spacing: Discourage mobbing by enforcing friendly fire and rewarding spatial awareness.
  • Favor Defenders: Prioritize defensive actions for players under multi-opponent assault.
  • Balance Group Combat: Manage simultaneous hits effectively, supporting puppet and minion archetypes while scaling to MMORPG potential.
  • Integrate Seamlessly: Enhance attack type interactions and maintain smooth pacing within the core gameplay loop.

Overview

Attack and damage Priority in The Fighting Game determines how simultaneous attacks resolve and how damage is distributed across players in a 3rd-person over-the-shoulder perspective, designed with group combat and puppet or minion archetypes as a core focus. Friendly fire remains always active, compelling players to consider their positioning relative to both allies and enemies. A rock-paper-scissors priority order, such as Grab prevailing over Block, and robust defender mechanics, like Absolute Guard with Throw Invulnerability, provide advantages to those under heavy attack. A damage buffer collaborates with damage decay to mitigate mobbing by spreading and reducing multi-opponent damage, ensuring tactical depth persists whether in 1v1 duels or chaotic multi-foe engagements.

Core Attack and damage Priority System

Priority and damage mechanics operate as follows:

  • Friendly Fire: Every attack, possessing an area of effect (applicable to almost all attacks), can strike friends, foes, and minions alike, encouraging careful spacing among players.
  • Attack Priority Order: A rock-paper-scissors system resolves overlapping attacks:
    • Grab: Prevails over Block if the target is within range, though Strikes can counter it during the Grab’s startup by hitting the Grabber with their attack’s area of effect; once locked, Grabs enable allied follow-up attacks through Combo Specials.
    • Control: Overrides other attacks unless interrupted during its startup or blocked if lockable—specific blockability details vary and are defined in the "control skills" section.
    • Beam: Overcomes Projectile and Strike attacks by delaying their impact with its buildup, but control skills or Grabs can interrupt it before completion.
    • Projectile: Can outrange Strike attacks, though timing determines what hits first—attacks with invincible elements may bypass Projectiles in some cases.
    • Strike: Interrupts Grab by striking the Grabber with its area of effect during startup, but other attack types generally supersede it unless two Strikes clash, resulting in mutual nullification with a 17-frame hitstop.
  • Defender Priority:
    • Defensive actions—including Instant Blocks, Parries, Deflects, Reflects, Combo Breaks, and Shoves—become easier to execute against multiple simultaneous hits, offering area-of-effect responses such as a Shove pushing opponents back 2 meters; exact boosts will be refined through prototyping.
    • Absolute Guard: Allows a defender to block all High, Low, and Mid simultaneous attacks within a blockstun chain, provided the first attack is successfully blocked.
    • Throw Invulnerability: Prevents characters from being thrown in specific situations to avoid infinite disadvantage states, including during prejump, blockstun plus 5 frames after, hitstun plus 5 frames after, knockdown plus 5 frames after, or when a move grants throw invulnerability or other relevant invulnerability.
  • Damage Buffer: Captures additional hits from multiple opponents within a 3-4 frame window and distributes the damage over 0.5 to 1 second, working in tandem with "Damage Decay" to reduce both damage and hitstun based on attack succession and type variety.

Integration with Combat

This system integrates with other combat elements to enhance gameplay:

  • Attack Types: The priority order—Grab over Control Skill, Control Skill over Beam, Beam over Projectile, and Projectile over Strike—ensures consistent resolution across interactions; for instance, a Grab overcomes a Block, while a Strike can counter the Grab by hitting its initiator with area of effect, and Beam clashes using Poise bids remain secondary to this order.
  • Attack Properties: Effects like Slash combined with Poison have their damage over time buffered and reduced through decay, preventing immediate stacking and spreading the impact over the distribution period.
  • Weapons: Longswords deliver Strike attacks and spears deliver Pierce attacks, both adhering to the priority order—skills can modify these interactions, for example, by adding a Fire burst to enhance their effect beyond basic damage.
  • Core Loop: Friendly fire challenges players to maintain advantageous spacing during Neutral, priority aids in anticipating outcomes during Prediction (e.g., a Grab defeating a Block), and the buffer paired with decay supports recovery in Disadvantage; 2v1 scenarios involving Grab and blockstun are flagged for prototyping to ensure balance and breakout opportunities.
  • Guard: Absolute Guard blocks multiple attack angles within a blockstun chain but does not stop Grabs, while defender boosts enhance mechanics like Instant Blocks, making them more effective against several simultaneous hits—Throw Invulnerability further protects against Grab loops.
  • Damage Decay: The buffer collaborates with damage decay, reducing the damage and hitstun of successive hits from multiple opponents over the distribution period, such as lessening the impact of three rapid Strikes to encourage varied attack strategies.

Challenges and Solutions

  • Grab Priority: The unblockable nature of Grabs from "Attack Types" contrasts with their counterability in "Normal Attacks."

    Solution: Grabs take priority over Block but can be countered by a Strike’s area of effect during the Grab’s startup—prototyping with a Shoto character’s Grab versus Strike will validate this balance and ensure counterplay remains viable.

  • Buffer and Decay Balance: The combined effect of the buffer and decay might overly weaken multi-opponent damage scenarios.

    Solution: Prototype a Shoto character facing three simultaneous Strikes to assess the buffer and decay interaction, adjusting the decay rate if the reduction proves too strong, particularly when diverse attack types are involved.

  • control skill Integration: Blockability details for control skills remain unspecified in this section to avoid overlap.

    Solution: Defer these specifics to the "control skills" section, noting only Control’s place in the priority order here, with prototyping to confirm its balance against a Shoto control skill.

Conclusions

Attack and damage Priority fosters tactical spacing through persistent friendly fire, prioritizes defenders with a rock-paper-scissors order (e.g., Grab overcomes Block, Strike counters Grab during startup), and balances group combat with a buffer and damage decay to reduce multi-opponent impact. Prototyping-ready for a Shoto character, such as testing a Strike against a Grab, it resolves priority conflicts, integrates smoothly with "Attack Types," "Attack Properties," "Guard," and "Damage Decay," and scales effectively while addressing 2v1 lock concerns through buffer, decay, Throw Invulnerability, and area-of-effect counters, enhancing the core loop’s strategic depth.

Normal Attacks

Goals

The normal attack system aims to:

  • Establish Core Combat: Provide accessible, state-driven basic attacks for all players.
  • Enhance Tactical Variety: Offer light, heavy, and grab options tailored to different combat scenarios.
  • Leverage State Dynamics: Utilize Directional and Aimed states to create distinct attack behaviors.
  • Support Customization: Enable flexible moveset design through normals and command normals.

Overview

Normal Attacks in The Fighting Game form the foundation of combat, executed through button presses with variations determined by the character’s current state—standing, crouching, jumping, or proximity to an opponent. Light attacks prioritize speed over power, heavy attacks deliver strong but slower blows, and grab attacks initiate a grapple state with counterplay potential. Command normals extend this system by combining directional inputs with button presses, offering additional tactical options. These attacks balance simplicity and depth, supporting both 1v1 duels and group combat scenarios.

Core Normal Attacks System

Normal attacks vary by type and state as follows:

  • Normals: Performed by pressing an attack button, adapting to the character’s state:
    • Light Attacks: Available in Base, Focus, Grapple, and Guard states, these deliver fast but weak strikes, ideal for quick engagements, typically targeting high, mid, or low attack spaces with consistent hitstun durations readable by opponents—light attacks generally feature lower attack levels for reduced impact compared to heavier options.
    • Heavy Attacks: Available in Base, Focus, Impact (via pressing the Light attack button after entering Impact), Grapple, and Guard states, these unleash slow yet powerful blows, suited for high, mid, or low attack spaces, maximizing damage at the cost of speed, with hitstun durations uniform across spaces and visually reduced by armor effects observable to players—heavy attacks typically boast increased attack levels for greater effect; in Impact state, entered by holding the Heavy attack button, the Light button triggers an Impact Heavy attack focused on knockup or knockdown effects, distinct from typical light attacks despite the shared input.
    • Grab Attacks: Available only in Base state, these initiate a grapple state to lock down opponents briefly, counterable by opposing attacks during startup, focusing on mid attack space to control foes—grapple duration varies by character design, with specifics detailed in "Grab Attacks."
    • Grappling Attacks: Function within the grapple state like Base state attacks, allowing directional throws or attacks that connect the player and opponent, delivering damage or repositioning based on the chosen follow-up after the initial grapple lock.
    • Control Attacks: Available only in Impact state (via pressing the Grab button after entering Impact), these deliver crowd control-focused effects at a high cost to Energy or Poise, distinct from grapple-based control despite using the Grab button—designed to disrupt rather than connect players like Grabs.
    • Attacks can be classified as high, mid, or low, influencing defensive responses without altering base hitstun expectations, which remain uniform across spaces with visual reductions from armor.
  • Command Normals: Executed by pressing a specific direction and an attack button simultaneously (e.g., Forward + Light), available across all states—Base, Focus, Impact, and others—with each state featuring a full set of normals as defaults; they require a single press rather than a hold, distinguishing them from charged attacks (specials), and do not support negative edge inputs, potentially overridden by charged or special attack inputs due to their distinct activation mechanics—total moveset slots are unlimited but capped by a value system based on factors like attack space, startup/recovery time, attack level, and additional effects (e.g., Cold or Bleed), with restrictions on mixing high-value attacks to balance archetypes, refined through prototyping.
  • States:
    • Directional States (Base, Grappling): Restrict attacks to normals and command normals, aligning their direction with movement controls (WASD or Left Thumbstick) for aggressive combat; forward-facing normals and command normals can extend to long-range distances depending on the character’s effective range and the player’s forward-facing view, while side or back directions limit range to close-to-mid distances with strong pushback effects in most cases—pushback can counter incoming Projectiles when properly timed, aiding multi-opponent control; unspecified command normal slots default to attacks aligned with the equipped weapon type (e.g., a longsword defaults to a Slash attack).
    • Aimed States (Focus, Impact, Guard): Limit command normals to forward-facing attacks, allowing any of the four core directional inputs to apply unique effects—for example, a Back + Heavy command normal might launch a forward spike from the ground while the character hops backward, maintaining the aimed direction with the implied movement from the input; unspecified directional slots default to the state’s normal attack (e.g., Back + Light empty defaults to the Light normal)—Impact state, entered by holding the Heavy attack button, uses the Light button for Impact Heavy attacks and the Grab button for Control attacks.

Integration with Combat

Normal attacks integrate with other combat systems to enhance gameplay:

  • Attack Types: Light and heavy attacks align with Strikes, Projectiles, and Beams, delivering direct damage tailored to their state, while grab attacks correspond to Grabs, initiating a grapple state, and control attacks in Impact state align with Control effects—command normals incorporate diverse types, such as a Forward + Light launching a Projectile in Base state or a Side + Heavy delivering a pushback effect from Base state to counter ranged threats from multiple angles.
  • Controls: Base state utilizes thumbstick directional inputs to align attacks with movement, enabling long-range forward attacks and close-to-mid side/back options with pushback, while Aimed states (Focus, Impact, Guard) leverage Held States for forward precision—Impact state, entered by holding the Heavy attack button, repurposes the Light button for Impact Heavy attacks (knockup/knockdown) and the Grab button for Control attacks, with charged attacks shifting from Base to Aimed via a held input, distinct from the immediate press required for command normals.
  • Attack Properties: Normals reflect weapon-specific properties, such as Slash for longswords or Pierce for spears, with skills defining and potentially enhancing them—for instance, adding a Poison effect to a heavy slashing Strike to apply damage over time, applicable across all ranges from close to long without altering hitstun expectations.
  • Attack Levels: Normals typically scale in speed and damage relative to command normals, specials, and supers, with light attacks featuring lower attack levels and heavy attacks higher ones—Impact Heavy and Control attacks further emphasize higher levels with increased impact or cost, with exact balance refined through prototyping to align with hitstun and damage expectations.
  • Attack and damage Priority: Grab attacks take priority over Block but remain vulnerable to Strikes interrupting their startup with area-of-effect damage; Guard state blocks normal attacks that are not Grabs or Control attacks within a blockstun chain, while Throw Invulnerability prevents Grab loops post-blockstun or hitstun for 5 frames—side/back pushback in Base state enhances multi-opponent defense by countering Projectiles when timed correctly.
  • Core Loop: Attacks probe Neutral by poking with rapid strikes or applying long-range pressure, secure Advantage with powerful hits or well-timed counters, and shift Prediction by locking foes with grabs or breaking guard—Base state’s long-range forward and close-to-mid side/back options with pushback support group combat spacing alongside friendly fire dynamics, ensuring versatility against multiple opponents.
  • Weapons: Equipped weapons define default command normals in Directional states if slots are unspecified (e.g., a longsword defaults to Slash attacks across directional inputs), while Aimed states revert to basic Strike normals when directional slots are empty, ensuring all inputs remain functional regardless of customization—weapon choice influences value-based caps on command normal selection.

Challenges and Solutions

  • Grab Definition: The original "control/grab" phrasing risked conflating Grab (grapple state) with Control (CC) from "Attack Types," potentially confusing their distinct roles in combat.

    Solution: Define grab attacks as initiating a grapple state from Base state, counterable during startup per "Attack and damage Priority," with duration and breakout mechanics deferred to a "Grab Attacks" section—prototyping with a character’s grab versus a Strike will confirm this balance maintains counterplay without undermining its priority over Block.

  • Command Normal Flexibility: Unlimited slots capped by a total value system (e.g., space, startup, attack level, effects) ensures all inputs produce attacks, but balancing weak utility versus strong key attacks with restrictions on high-value mixes must align with archetype diversity.

    Solution: Implement Directional command normals defaulting to weapon-aligned attacks (e.g., longsword Slash) and Aimed states reverting to normals (e.g., Light attack) when unspecified—prototyping will assess if this value-based cap supports sufficient tactical variety and archetype balance, adjusting restrictions as needed.

  • Impact State Clarity: Impact state’s input scheme (Heavy hold to enter, Light for Impact Heavy, Grab for Control) may confuse players expecting traditional Light or Grab behavior, overlapping with "Controls" Held State intent.

    Solution: Clearly define Impact state entry via holding the Heavy attack button, with Light triggering Impact Heavy (knockup/knockdown) and Grab triggering Control attacks—prototyping will test if these repurposed inputs are intuitive and distinct from Base state actions, ensuring clarity against multi-opponent scenarios.

  • Pushback Utility: Side/back pushback countering Projectiles introduces a mechanic not explicit in "Attack and damage Priority," where Projectile beats Strike, potentially shifting priority dynamics.

    Solution: Frame pushback as a timing-based counter (e.g., deflecting Projectiles when aligned with the command normal’s execution), ensuring it complements rather than overrides the established priority order—prototyping will validate this against a character facing multiple ranged foes, using examples like a Side + Heavy command normal, to ensure it enhances defense without disrupting balance.

Conclusions

Normal Attacks deliver a robust, state-driven combat foundation, balancing light, heavy, grab, and control options with command normals to offer tactical variety across builds. With Directional states supporting long-range forward attacks and close-to-mid side/back options featuring pushback to counter Projectiles, and Aimed states providing forward-facing directional flexibility with Impact-specific Heavy and Control attacks, they resolve conflicts like grab counterability, integrate seamlessly with "Attack Types," "Controls," "Attack Properties," "Attack Levels," and "Attack and damage Priority," and scale effectively for group combat through range dynamics, pushback utility, and friendly fire synergy—enhancing the core loop’s strategic flow while supporting predictable hit reactions and value-based customizable movesets.

Charged Attacks

Charged attacks are attacks that require an attack key be held to Charge the attack up before release. This usually produces an attack that can scale in damage based on the duration the attack is held for. Additional scaling occurs when used as part of a combo.

Charged attacks could be considered more powerful than specials. Due to the way specials are set up charges may necessarily replace a set of specials but that would depend on if it feels better to lock in the charge or have it just be part of the structure of specials.

Channeled attacks and skills

Channeled attacks would be a sub set of Charged attacks with the difference being the attack comes out continuously until the button is released.

Special Attacks

Attack button + Motion input

Special attacks are fancier attacks that have special properties such as invincibility or unique movement. Specials can only by done with negative edge so the attack button must be held while the motion input is applied.

There are two main categories of special move inputs:

  • Radial: Requires the player to input directions in a quarter or half circle motion, then input a specific attack button.
  • Zig-Zag inputs: Requires the player to input one direction, then input the opposite direction plus a specific attack button.

There are a few common qualities that special moves share:

  • They do 5% of their base damage as chip damage on block unless the defender uses Faultless Defense or Instant Block.
  • They build a little bit of Poise immediately when inputted.
  • They can be kara canceled into Supers during the being of their startup animation

Because specials require negative edge the attack button is assumed to be held at least for a brief duration. Potentially longer duration presses would transition to charged attacks. If this feels bad, it may be preferable to specify some attack states as charges or specials only (slash state vs base state for example)

Super Attacks

Motion input + attack button. Costs 50% (or 100%) Poise.

Supers are attacks that are even more powerful than special attacks.

Some Supers are useful as reversals, while others power up the character. On start-up, Supers freeze the game. This is commonly referred to as "super flash" or "super freeze". Super freeze also has a special input buffer making these attacks not a good "surpise" attack since they easily reactable during neutral.

Combo Special

There are currently two Combo keys defined by the 2 otherwise unused face buttons on the gamepad (x and y) The idea with these is to work exclusively as situational attacks that are only available under certain circumstances or when certain requirements are met. This could be a way to apply Crowd control after other skills prime it. or follow up certain attacks with a thematic supporting attack. This is directional so targets that meet the conditions would be indicated and could be hit with these attacks quickly.

Combo Break

While in hitstun a combo break is performed by holding block and pressing either light or heavy attack. You must match the light or heavy type of the incoming attack. If the player selects a non-matching attack type they end up in an extended hitstun and cannot attempt to perform a combo break again for some amount of time either related to the hitstun.

As this is a form of counter-attack this could possibly also be used to break a grab or control attack during a combo but in these cases matching with the grab button is not needed as attacks beat grabs already. This may be unnecessary clarification as attacks don’t normally link or combo into grabs or control but if edge cases exist this covers it.
